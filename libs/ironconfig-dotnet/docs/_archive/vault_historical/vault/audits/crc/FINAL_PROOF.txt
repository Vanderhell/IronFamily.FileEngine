A) CRC Computation Order
   - Header is written with correct crcOffset BEFORE CRC computation
   - CRC is computed over header+data (including finalized header)
   - This ensures header with crcOffset field is included in CRC
   - Prevents circular dependency issues

B) Endianness (Little-Endian)
   - CRC stored as 4 bytes: byte0=LSB, byte3=MSB
   - Encoding:   Add bytes (crc & 0xFF), ... ((crc >> 24) & 0xFF)
   - Decoding:   data[0] | (data[1]<<8) | (data[2]<<16) | (data[3]<<24)
   - Test vector proves correct interpretation

   - CRC computed over correct byte range
