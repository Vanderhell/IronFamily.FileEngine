10. CRITICAL DESIGN DECISIONS
==============================

âś“ EMPTY sentinel = 0xFFFFFFFF:
  - 4294967295 when encoded as VarUInt uses 5 bytes
  - Doesn't collide with valid pairIndex (0 to count-1, max 1M)
  - Canonical representation

âś“ Table sizing = nextPow2(pairCount * 2):
  - Ensures load factor â‰¤ 0.5
  - Minimum 8 slots (safety margin)
  - Power-of-2 enables bitmask modulo operation

âś“ Knuth multiplicative hash = 2654435761:
  - Golden ratio approximation
  - Proven good distribution properties
  - Same constant used in other high-quality hash functions

âś“ Storage layout (pairs first, table second):
  - Allows iterating all pairs by reading from start
  - Hash table is optional secondary lookup
  - Enumerator doesn't pay hash table cost

âś“ Zero-copy design preserved:
  - No heap allocations required for indexed lookup
  - Struct-based views (no GC pressure)
  - Direct buffer reading via offsets
