KEY DESIGN DECISIONS
================================================================================

1. byte[] Over ReadOnlySpan<Byte>:
   ReadOnlySpan<T> cannot be stored in struct/class fields (C# limitation).
   Traded pure zero-copy for practical, working implementation.
   Trade-off: Minimal allocations only at parse phase (dictionary, VSP);
              value reads remain allocation-free.

2. Offset-Based Value Indexing:
   Used relative offsets (VarUInt-encoded) for ARRAY and OBJECT elements.
   Enables zero-copy navigation within payload.
   Index table support: Outlined for v0; can be added for O(1) object lookup.

3. CRC32 as Optional Trailer:
   Matches BJV approach; IEEE 802.3 CRC32 over bytes [0 .. crc_offset).
   Set via --crc on flag during packx.
   Validated automatically in IcfxView constructor.

4. Determinism via Canonical Rules:
   Same rules as BJV: sorted dictionaries, lexicographic ordering, etc.
   Enforced in IcfxEncoder via StringCollector + sorting.
