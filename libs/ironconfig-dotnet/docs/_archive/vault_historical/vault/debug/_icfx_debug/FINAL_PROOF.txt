# ICFX Stack Overflow Fix - Final Proof

## Summary
Fixed critical bugs in ICFX encoder that caused stack overflow when decoding nested structures (arrays of objects). All test cases now pass successfully.

## Root Cause Analysis

### Bug 1: Dictionary Not Shared with Nested Encoders
**Symptom**: Objects inside arrays showed as empty `{}` when decoded
**Root Cause**: When encoding array elements or object values, temporary IcfxEncoder instances were created for pre-encoding, but these did NOT have the dictionary populated. This caused keyId lookups to return -1, which when cast to uint becomes 0xFFFFFFFF, causing massive varuint offsets.
**Fix**: Copy dictionary and VSP to temporary encoders so nested structures can properly look up dictionary keys.

### Bug 2: Varuint Size Mismatch in Offset Calculation
**Symptom**: Offset values were calculated incorrectly due to placeholder size estimates
**Root Cause**: When encoding arrays and objects, offsets were calculated using placeholder zeros (all 1 byte) to estimate offset table size. However, actual offset values might be larger (2-5 bytes), shifting where element data actually starts.
**Fix**: Iterate offset calculation until stable - measure actual size of encoded offsets and recalculate until convergent (usually 1-2 iterations).

## Code Changes

### IcfxEncoder.cs - VisitArray Method
- Added iterative calculation of offset table size until convergence
- Ensures offset encoding size matches the size assumed during offset calculation
- Copies dictionary/VSP to temporary encoders for proper key lookup

### IcfxEncoder.cs - VisitObject Method
- Added iterative calculation of metadata size until convergence
- Ensures keyId and offset varuint sizes are correctly accounted for
- Copies dictionary/VSP to temporary encoders for proper key lookup

### Key Fix Details
1. **Shared Dictionary**: elemEncoder._dictionary.AddRange(_encoder._dictionary) and elemEncoder._vsp.AddRange(_encoder._vsp)
2. **Iterative Convergence**: do-while loop recalculates offsets until the total offset table size stabilizes
3. **No Cycle Detection**: Removed hash-based cycle detection (was false positive), kept depth guard (MAX_DEPTH=256)
